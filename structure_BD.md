# Users (пользователи) Таблица, хранит информацию о зарегистиррованных пользователях
 Поля таблицы:
 id user (INT)
 логин (CHAR(n))
 пароль (CHAR(n))
 дата регистрации (DATE)
 e-mail (CHAR(n))
 телефон (INT)

 # Content. В этой таблице будут храниться отредактированные после LLM тексты пользователей.
 id user - многие к одному # У одного юзера несколькр контентов
 Исходный текст (ограничение 2000 символов, хранится только в локальном хранилище браузера)
 Язык введенного текста (русский, китайский, английский, испанский, автораспознование языка текста)
 Исправленный текст (хранится в БД)

 обращения в поддержку (сообщения)

 # Correction (исправление)
 id user (INT)
 Тексты после исправления (VARCHAR(n))
 дата и время исправления (DATETIME)
 Тип исправления(орфография, пунктуация, все типы) (CHAR(n))
 Тексты после обработки ИИ (VARCHAR(n))

 # Связь между Users, Content, Correction
 User - Content: один пользователь может создать несколько документов
 Content - Correction: один текст может быть исправлен несколькими способами
 Correction - User: исправление пренадлежит пользователю. Количество исправлений одного текста одним пользователем и даты вносимых исправлений
